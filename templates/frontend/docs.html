All Users (GET)

This endpoint provides access to the data of all users stored in the database, returning a JSON response that includes key details for each user, such as their unique user ID, full name, email address, mobile number, city, and encrypted password. By making a request to this endpoint, you can retrieve a comprehensive list of all users in the database. This data can be used for various purposes, including user management and auditing. However, it is important to ensure that access to this endpoint is properly secured to protect sensitive information, such as passwords and personal contact details.

http://127.0.0.1:8000/api/users

Filtered Users Data (GET)

This endpoint allows you to retrieve a filtered list of users based on specific criteria. You can specify any field, such as name, email, or mobile number, to filter the users accordingly. By providing the desired filter parameters in the request, the API will return a list of users that match the given conditions. This feature enables more targeted queries, making it easier to find users based on particular attributes. It is important to ensure that the filters are properly validated to prevent unnecessary data exposure.

http://127.0.0.1:8000/api/users/?city='Agra'

Specific User EndPoint (GET)

This endpoint provides detailed information about a specific user. By hitting the API and appending the user ID to the request, you will receive a JSON object containing the user's data, including their unique user ID, full name, email address, mobile number, and encrypted password. This allows you to retrieve specific information for individual users. It is important to ensure proper authentication and authorization when accessing this endpoint to protect the privacy and security of user data.

http://127.0.0.1:8000/api/users/2
